# 数式エディタ (Mathchaライク) 設計書

## 1. 概要
本設計書は、Mathcha (https://www.mathcha.io/) のような直感的な操作感を参考にしつつ、現在のMarkdown+KaTeXベースのアプリケーションに適合する「インライン数式エディタ (WYSIWYGライク)」の設計を定義する。ユーザーからの要望に基づき、フルスクラッチのWYSIWYGエディタではなく、プレビュー上の数式を直接編集しているように見せる軽量なアプローチを採用する。

## 2. コア機能要件

Mathchaの機能調査に基づき、以下の主要機能を提供する。

### 2.1. テキスト＆数式入力 (WYSIWYGライク)
- **テキストモード / 数式モードのシームレスな統合**:
  - 行ベースの段落、インデント、リスト（番号付き/箇条書き）
  - テキスト中に `$` などをトリガーとしてインライン数式 (`\inline-math`) を挿入
  - 独立したブロック数式 (`\math-container`) を挿入
- **サジェストボックス (オートコンプリート)**:
  - `\` (バックスラッシュ) をトリガーとして展開される高機能サジェストUI
  - シンボルの名前検索、カテゴリ別検索、手書き認識（Draw to Find）による検索
- **自動フォーマット＆ショートカット**:
  - `^` で上付き文字 (指数)、`_` で下付き文字 (添字) の自動展開
  - 括弧の自動ペアリング・高さ自動調整 (`\left`, `\right` 相当の自動解釈)
- **インラインKaTeXエディタ (疑似WYSIWYG)**:
  - **トリガー**: プレビューペイン内でレンダリングされたKaTeX数式（`.katex-html`や`.katex`要素）をクリックする。
  - **編集UI**: クリックされた数式の上にオーバーレイでテキスト入力フィールド（または小型のCodeMirrorインスタンス）を表示する。
  - **リアルタイムプレビュー**: 入力フィールドでLaTeXコードを編集すると、その場で即座に（あるいは入力欄のすぐ下で）変更後の数式がレンダリングされる。
  - **双方向同期**: 編集を完了（フォーカスが外れる、またはEnterキー押下）すると、元のMarkdownエディタ（CodeMirror）内の該当するKaTeXコードブロック（`$$...$$` や `$...$`）のテキストが自動的に置換・更新される。
  - **ユーザー体験**: ユーザーからは「プレビュー上の数式をクリックして直接修正し、即座に反映される」ように感じられ、実態としては裏側のテキストコードを書き換えている形となる。

### 2.2. 高度な数式レイアウト
- **行列 (Matrix) / 配列 (Array) / 条件分岐 (Cases)**:
  - グリッドUIを利用した行・列の追加・削除（右クリックコンテキストメニュー）
  - 括弧のスタイル変更 (丸括弧、角括弧、波括弧など)
- **数式アライメント**:
  - `Aligned`, `Gathered`, `Multiline` モードを用いた、複数行数式の等号位置揃えなど
- **タグ/参照**:
  - 数式に番号タグ (例: `(1)`) を付け、ドキュメント内の別場所からリンク参照する機能

### 2.3. 図形描画エリア (Drawing Area)
- **グリッド＆スナップ**:
  - ドキュメント内にキャンバス領域を挿入
- **コンポーネント**:
  - テキストボックス（境界線、背景色変更、数式埋め込み可能）
  - 接続線、矢印（直線、曲線、制御点付き、Commutative Diagram用）
  - 各種図形（円、楕円、多角形、波線）
  - 座標軸、グラフの描画コンポーネント

### 2.4. エクスポート・インポート
- LaTeXコードからのインポート
- 選択範囲またはドキュメント全体のLaTeXコード出力
- 数式や描画エリアの画像 (PNG / SVG) エクスポート機能
- PDF印刷出力対応

## 3. アーキテクチャ・技術スタック設計

WYSIWYGの数式エディタは、テキストエディタの中でも最も複雑な部類に入るため、状態管理とレンダリングの分離が重要。

### 3.1. フロントエンド技術
- **フレームワーク**: 既存のVanilla JSベースのコードに統合。
- **既存UIとの統合**:
  - 現在のプレビューペイン (`src/js/preview_inline_edit.js` など) のイベントリスナーを拡張し、数式要素 (`.katex`) のクリックを検知する。
- **データ構造 (Model)**:
  - ドキュメント全体は引き続きMarkdownテキストとして CodeMirror 6 で管理する。
  - プレビュー上でクリックされた要素と、CodeMirror内の行番号・文字位置をマッピングする仕組み（`data-line` 属性や数式の生テキストマッチング）を活用する。
- **数式レンダリング**:
  - 現在の `markdown-it-texmath` および `katex` の仕組みを流用。
  - インラインエディタのリアルタイムプレビュー用には `katex.render()` または `katex.renderToString()` を直接呼び出す。

### 3.2. サジェスト・手書き認識エンジン
- **手書き認識**:
  - クライアントサイドでの機械学習モデル (TensorFlow.js などを利用し、MNISTライクな数式シンボルデータセットで学習済みのモデルをブラウザ上で実行)、もしくはMyScript等の外部API利用。

### 3.3. データ管理と保存
- LocalStorage / IndexedDB を用いたオフライン保存機能
- クラウド同期 (バックエンド: Node.js + MongoDB または Firebase)

## 4. 開発のフェーズ分け

既存アプリケーションへの統合を考慮し、以下のステップで進める。

* **Phase 1: インラインエディタの基盤構築**
  - プレビュー上のKaTeX要素クリック検知の追加。
  - オーバーレイ用テキスト入力UIの表示と、元のMarkdownテキスト内の該当LaTeXコードの抽出。
* **Phase 2: リアルタイムプレビューと同期**
  - テキスト入力中のリアルタイムKaTeXレンダリングの実装。
  - 編集確定時にCodeMirrorインスタンスへ変更を反映（ディスパッチ）し、プレビューを自動更新する機能の追加。
* **Phase 3: 高度なUI/UX向上**
  - `\` トリガーによるLaTeXコマンドのサジェストボックスの追加。
  - 手書き認識エンジンとの連携（Draw to Find機能）。
