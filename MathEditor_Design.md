# 数式エディタ (Mathchaライク) 設計書

## 1. 概要
本設計書は、Mathcha (https://www.mathcha.io/) のような、直感的でWYSIWYG（見たままが得られる）な操作感を持つ高度な数式・ドキュメントエディタのコア機能およびアーキテクチャ設計を定義する。

## 2. コア機能要件

Mathchaの機能調査に基づき、以下の主要機能を提供する。

### 2.1. テキスト＆数式入力 (WYSIWYG)
- **テキストモード / 数式モードのシームレスな統合**:
  - 行ベースの段落、インデント、リスト（番号付き/箇条書き）
  - テキスト中に `$` などをトリガーとしてインライン数式 (`\inline-math`) を挿入
  - 独立したブロック数式 (`\math-container`) を挿入
- **サジェストボックス (オートコンプリート)**:
  - `\` (バックスラッシュ) をトリガーとして展開される高機能サジェストUI
  - シンボルの名前検索、カテゴリ別検索、手書き認識（Draw to Find）による検索
- **自動フォーマット＆ショートカット**:
  - `^` で上付き文字 (指数)、`_` で下付き文字 (添字) の自動展開
  - 括弧の自動ペアリング・高さ自動調整 (`\left`, `\right` 相当の自動解釈)
  - 分数 (`\frac`) や行列などの入力時に、専用のUIブロックに変換し、カーソル移動で分母・分子を行き来可能にする

### 2.2. 高度な数式レイアウト
- **行列 (Matrix) / 配列 (Array) / 条件分岐 (Cases)**:
  - グリッドUIを利用した行・列の追加・削除（右クリックコンテキストメニュー）
  - 括弧のスタイル変更 (丸括弧、角括弧、波括弧など)
- **数式アライメント**:
  - `Aligned`, `Gathered`, `Multiline` モードを用いた、複数行数式の等号位置揃えなど
- **タグ/参照**:
  - 数式に番号タグ (例: `(1)`) を付け、ドキュメント内の別場所からリンク参照する機能

### 2.3. 図形描画エリア (Drawing Area)
- **グリッド＆スナップ**:
  - ドキュメント内にキャンバス領域を挿入
- **コンポーネント**:
  - テキストボックス（境界線、背景色変更、数式埋め込み可能）
  - 接続線、矢印（直線、曲線、制御点付き、Commutative Diagram用）
  - 各種図形（円、楕円、多角形、波線）
  - 座標軸、グラフの描画コンポーネント

### 2.4. エクスポート・インポート
- LaTeXコードからのインポート
- 選択範囲またはドキュメント全体のLaTeXコード出力
- 数式や描画エリアの画像 (PNG / SVG) エクスポート機能
- PDF印刷出力対応

## 3. アーキテクチャ・技術スタック設計

WYSIWYGの数式エディタは、テキストエディタの中でも最も複雑な部類に入るため、状態管理とレンダリングの分離が重要。

### 3.1. フロントエンド技術
- **フレームワーク**: React (または Vue.js)
  - DOMツリーの仮想DOMによる状態管理と、再描画の最適化のため。
- **データ構造 (Model)**:
  - ProseMirror または Slate.js のようなカスタムエディタフレームワークの基盤を利用。
  - 単なるプレーンテキストやMarkdownではなく、JSONツリー構造 (AST: Abstract Syntax Tree) でドキュメントを保持。
  - 例: `[ { type: 'paragraph', children: [{ text: 'Solve for x: ' }, { type: 'math-inline', latex: 'x^2 = 4' }] } ]`
- **数式レンダリング**:
  - KaTeX (非常に高速でDOMへの組み込みが容易。MathJax 3も候補)
  - エディタ上で編集可能な数式ブロックは、単なる画像ではなく、入力用の `<input>` や `contenteditable` 要素をHTMLとCSSで数式の形に組み上げた「エディタコンポーネント」として実装する。
- **図形描画 (Drawing)**:
  - SVG と D3.js または Paper.js / Fabric.js
  - 数式の埋め込みとスケーラビリティを考慮するとSVGベースの描画が最適。

### 3.2. サジェスト・手書き認識エンジン
- **手書き認識**:
  - クライアントサイドでの機械学習モデル (TensorFlow.js などを利用し、MNISTライクな数式シンボルデータセットで学習済みのモデルをブラウザ上で実行)、もしくはMyScript等の外部API利用。

### 3.3. データ管理と保存
- LocalStorage / IndexedDB を用いたオフライン保存機能
- クラウド同期 (バックエンド: Node.js + MongoDB または Firebase)

## 4. 開発のフェーズ分け

数式エディタは巨大なプロジェクトになるため、段階的な開発を推奨する。

* **Phase 1: コアエディタと基本数式**
  - Slate.js / ProseMirror の導入
  - テキスト入力と、ブロック/インラインの数式ノードの定義
  - KaTeXを用いた数式の静的レンダリング
* **Phase 2: WYSIWYG数式編集UI**
  - 数式ブロック内でのカーソル移動、分数や上付き/下付き文字をキーボードでシームレスに入力するカスタムUIの実装（ここが一番の難所）
  - サジェストボックス (ショートカット展開) の実装
* **Phase 3: 高度なレイアウトと図形描画**
  - 行列・テーブルエディタの実装
  - SVGキャンバスを用いた Drawing Area の実装と数式ノードの統合
* **Phase 4: インポート/エクスポート・手書き認識**
  - LaTeXパーサーの開発 (LaTeX -> JSON AST)
  - 手書きによるシンボル検索機能の導入
