# ペイントツールライブラリ (PaintLibrary) 仕様・設計書

## 1. 概要
本ツールは、HTML内に埋め込み、もしくはポップアップとして起動できる軽量で実用的なJavaScriptペイントツールライブラリです。  
独自の依存ファイルを含まず、完全オフラインで動作し、ブラウザの基本機能（Canvas API）を活用して設計されています。背景は透明色（PNG）をネイティブにサポートし、JPEG/PNGなどの汎用フォーマットでの保存・読み込みが可能です。

## 2. アーキテクチャ設計

### 2.1. デュアルキャンバス（オーバーレイ）構造
描画性能と柔軟なツール挙動（プレビューやフローティング）を実現するため、ライブラリ内部でキャンバスを2層構成にしています。
- **メインキャンバス (`this.canvas`)**: 確定した描画情報や読み込んだ画像が保持される本番用レイヤー。
- **オーバーレイキャンバス (`this.overlayCanvas`)**: メインキャンバスの上に位置し、マウスドラッグ中の「図形の枠線プレビュー」「矩形選択の点線」「コピペ時の移動可能なフローティング画像」など、確定前の情報を一時的に描画するためのレイヤー。
- **合成プロセス**: ユーザーがマウスを離した、あるいは別の場所をクリックした（確定アクション）タイミングで、オーバーレイCanvasの描画内容が `drawImage()` によってメインキャンバスへ焼き付けられ、Undo履歴に保存されます。

### 2.2. テキスト描画層の実装
キャンバスに直接文字を打つのではなく、DOMレイヤー（`<div contenteditable="true">`）をオーバーレイの上に重ねて配置します。入力確定時（フォーカスが外れた時等）に入力内容を取得し、Canvas API の `fillText` によってメインキャンバスに焼き付けるハイブリッド設計を採用しています。

## 3. 実装機能仕様

### 3.1. 描画・操作ツール
- **ブラシ**: 自由曲線描画。
- **消しゴム**: ブラシと同様の操作感で、透明色を塗る（`destination-out` を利用し背景を透過）。
- **塗りつぶし (Flood Fill)**: スタックベースのBFS（幅優先探索）アルゴリズムを用いた塗りつぶし。
- **図形描画**: 
  - 矩形（`rect()`）および 楕円（`ellipse()`）をドラッグして描画。
- **矢印描画**:
  - `arrow-up`, `arrow-down`, `arrow-left`, `arrow-right`
  - ドラッグした矩形の範囲に内接する「先端の三角形」と「軸の矩形」を組み合わせたベクターパスによる矢印を描画。
- **テキスト**: クリックした座標から文字を入力でき、色やフォントサイズ（太さに基づく）指定が反映される。

### 3.2. 選択・コピー・貼り付け
- **矩形選択**: ドラッグで点線の矩形領域を選択（`overlayCtx` に描画）。
- **コピーコマンド**: 選択状態の矩形座標をもとに、メインキャンバスから `getImageData` で画像情報を取り出し、ライブラリ内部メモリ（クリップボード変数）へ保持する。
- **貼り付けコマンド**: メモリの画像データで一時Canvas（`Image`相当）を生成し、オーバーレイキャンバス上の固定位置に「フローティング状態」として描画する。
  - フローティング中は、マウスドラッグで画像を自由に移動できる。
  - テキストや別のツールを使用、または枠外をクリックすると、決定された位置に焼き付けられ、確定する。

### 3.3. 編集・画像管理機能
- **Undo / Redo (履歴管理)**: 
  - メインキャンバスの状態が変わるアクション（描画確定、消去、貼り付け等）ごとに、現在の状態を DataURL (PNG) の配列として保存。
  - 履歴はメモリ管理の観点から最大30ステップまで保持する仕様。
- **画像の読み込み**: 
  - バイト列（Base64形式のDataURL）や、`File` オブジェクトを直接読み込み、画像のサイズに合わせてキャンバスに転写する。
- **画像の保存**: PNG形式（背景透過）とJPEG形式（透過部分は黒扱いになる等の標準仕様）でのエクスポート機能。
- **全消去**: キャンバス全体をリセット（clearRect）する機能。

## 4. インターフェース（API仕様）

### 4.1. 初期化オプション引数
```javascript
const paint = new PaintLibrary({
  container: document.getElementById('wrapper'), // 描画先DOM要素
  width: 700,
  height: 400,
  toolbar: true, // trueにすると内蔵ツールバーを自動追加
  onToolChange: (toolId) => { /* ツール変更時の処理 */ },
  onHistoryChange: (canUndo, canRedo) => { /* Undo可否変化時の処理 */ },
  onColorPickerRequest: (currentColor, anchorEl) => { /* カラーピッカー呼び出し時の処理 */ }
});
```

### 4.2. 利用可能ツールID (`setTool(tool)` に渡す引数)
- `brush`
- `eraser`
- `fill`
- `rect`
- `ellipse`
- `text`
- `select`
- `arrow-up` / `arrow-down` / `arrow-left` / `arrow-right`

### 4.3. 主なパブリックメソッド
| メソッド名 | 説明 |
|------------|------|
| `setTool(tool)` | アクティブなツールを変更する。フローティング状態などは強制的に確定される。 |
| `setColor(hex)` | 描画色（#RRGGBB など）を変更する。 |
| `setLineWidth(px)` | ブラシ太さ・図形の線幅・テキストのサイズなどに影響する。 |
| `copySelection()` | 矩形選択範囲を内部にコピーする。 |
| `paste()` | コピーした画像をキャンバス上に貼り付け（フローティング化）する。 |
| `undo()` / `redo()` | 履歴を戻る／進む。 |
| `clearCanvas()` | 全て消去。 |
| `loadImage(source)` | Base64 または `File` オブジェクトを読み込む（Promiseを返す）。 |
| `downloadImage(name, fmt)` | 画像としてダウンロード保存する。 |
| `resizeCanvas(w, h)` | 描画はクリアされるが、キャンバス全体の寸法を変更する。 |
| `getCanvas()` | 画像処理用にネイティブの HTMLCanvasElement 本体を返す。 |

---

## 5. 結合構成
本ツールは外部ライブラリを直接バンドルしていませんが、汎用カラーピッカー（`ColorPickerUI`など）と結合して動作することを前提としています。`onColorPickerRequest` コールバックを用いて外部ピッカーを開き、その `onChange` 等のイベントから `paint.setColor(hex)` を呼び出すことで密結合を回避しつつ、任意のカラーピッカープラグインとシームレスに連動します。
